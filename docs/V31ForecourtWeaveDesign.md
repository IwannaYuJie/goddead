# Goddead v31 设计：门前三岔 / THE FORECOURT WEAVE

## 目的

把最前段从「敲门 → 任意守则 → 走廊」的一条固定流程，改成玩家第一次打开网页就能发现的真实多分支。

- 新内容必须发生在门外与访客守则这一段，不挂在 v29/v30 深层房间之后。
- 门仍保留三次敲击后自动进入守则的主路径。
- 门图上的另外三个物件可以立即进入三个新区域；不是装饰，也不需要先解锁。
- 守则八条不再全部通往同一个地方，其中四条会通向三个新区域。
- 所有核心动作都是「点击或键盘激活 → 立即反馈 → 自动转场」，不增加第二个继续按钮。

## 现场问题

当前前段：

1. 门外只有整扇门是有效目标。
2. 第三次敲门会正确自动进入守则。
3. 守则有八个按钮，但第一次点击任意一条都会进入同一个 `#corridor`；视觉上像选择，实际是假分支。
4. 走廊已有 v29 三条支线，但要先主动点到指定残页才出现，而且位置已经晚于用户要求的「前面部分」。

本轮不改 v29/v30 内容，只把分支提前到第一次敲门之前和第一次阅读守则时。

## 新前段结构

```text
门外 #threshold
├─ 敲门三次 ───────────────→ 访客守则 #protocol
├─ 点击日蚀窥孔 ───────────→ 倒置窥孔 #peephole-chamber
├─ 点击左墙「不要数符号」 ─→ 失号龛 #glyph-niche
└─ 点击右侧「回来」痕迹 ───→ 回返夹道 #return-passage

访客守则 #protocol
├─ 其二 ─→ 回返夹道
├─ 其三 / 其七 ─→ 失号龛
├─ 其四 ─→ 倒置窥孔
└─ 其一 / 其五 / 其六 / 其八 ─→ 走廊
```

三个新区域之间还要形成三角回路，玩家可以在前段游走后再选择进入守则、走廊或回到门外。

## 门外三个新热点

沿用现有 `assets/threshold-bureau-door.webp`，不替换首页图片。

1. 日蚀窥孔
   - 位置：门上方正中的黑色日蚀。
   - 可访问名称：「观察门上的黑色日蚀」。
   - 去向：`#peephole-chamber`。
2. 左墙符号
   - 位置：左上「不要数符号」文字附近。
   - 可访问名称：「触碰左墙上不该数的符号」。
   - 去向：`#glyph-niche`。
3. 右侧回返痕
   - 位置：右侧「回来」文字与暗处。
   - 可访问名称：「沿着右侧写着回来的痕迹走」。
   - 去向：`#return-passage`。

三个热点都用原生 `button`，与门按钮同处一个相对定位容器；不得盖住门的主要敲击区域。默认只给极淡轮廓与短标签，hover/focus-visible 时增强，触屏上保持可发现。点击后立刻记为 visited，给 0.7–1.0 秒反馈后自动进入；reduced-motion 约 0.3 秒。

## 新区域一：倒置窥孔

- Hash：`#peephole-chamber`
- 中文名：倒置窥孔
- 英文：THE INVERTED PEEPHOLE
- 素材：
  - 源图：`design-references/source-forecourt-peephole.png`
  - 正式图：`assets/forecourt-peephole.webp`
- 视觉：从门内部看向一个巨大倒置窥孔；圆形黑镜居中，左侧是黄铜听音管，右侧有一只闭合的机械眼睑。黑金、暗红、档案机关质感，与现有门和走廊同一世界。

三个动作：

1. 「直视黑镜」
   - 记录 `witnessed`。
   - 反馈：「镜后那只眼，比你晚眨了一次。」
   - 自动去 `#protocol`。
2. 「听黄铜管」
   - 记录 `heardInside`。
   - 反馈：「敲门声从你身后传来。你明明还在门外。」
   - 自动去 `#return-passage`。
3. 「闭上这只眼」
   - 记录 `refusedSight`。
   - 反馈：「黑镜替你继续看着。」
   - 自动回 `#threshold`。

## 新区域二：失号龛

- Hash：`#glyph-niche`
- 中文名：失号龛
- 英文：THE MISSING-NUMERAL NICHE
- 素材：
  - 源图：`design-references/source-forecourt-glyph-niche.png`
  - 正式图：`assets/forecourt-glyph-niche.webp`
- 视觉：狭窄石龛里悬着九枚可转动的黄铜编号盘，其中第七枚被刮除，最底部有一枚完全空白的白瓷盘；墙面密布被反复擦掉的计数痕迹。

三个动作：

1. 「数第九道刻痕」
   - 记录 `countedNine`。
   - 反馈：「你数到九。墙里只回了八声。」
   - 自动去 `#peephole-chamber`。
2. 「擦掉第七号」
   - 记录 `erasedSeven`。
   - 反馈：「第七号消失后，守则多出一行空白。」
   - 自动去 `#protocol`。
3. 「取下空白号牌」
   - 记录 `tookBlank`。
   - 反馈：「没有编号的门，为你让出了一条路。」
   - 自动去 `#corridor`。

## 守则真实分流

保留现有八条守则文本、异常「玖」和第九条逻辑。

- 其二：取消原本去走廊的 AutoAdvance，短反馈后去 `#return-passage`。
- 其三：取消原 AutoAdvance，短反馈后去 `#glyph-niche`。
- 其四：取消原 AutoAdvance，短反馈后去 `#peephole-chamber`。
- 其七：保留现有第七条异常表现；普通激活时去 `#glyph-niche`。如果本次点中既有「玖」异常，仍优先进入原 `#ninth`，不得被 v31 改写。
- 其一、其五、其六、其八：保持原有短反馈后自动去 `#corridor`。

点击时先持久化选择，再调度转场。离开守则或新区域时必须取消未触发的旧 timer，避免回退后被幽灵转场拉走。

## 状态

独立键：`goddead_v31_forecourt_weave`

字段：

- `visited.peephole`
- `visited.glyph`
- `visited.returnPassage`
- `marks`：只允许九个合法动作标记，去重
- `lastChoice`：最后一次合法动作 ID
- `transitions`：非负整数，裁剪到安全上限

要求：

- 坏 JSON、错误类型、非法标记安全回退。
- 点击动作时立即保存，重复点击不重复写分，也不重复调度。
- 不读取、不覆盖 v28/v29/v30 和旧主线状态。
- 三个新区域不设解锁门槛；干净存档可从门外进入，也允许直接 hash。
- 重进时恢复最后选择的 `aria-pressed`，但所有动作仍可重新选择。

## 走廊、目录与痕迹

- 三个新区域各首次访问后，在目录出现入口；未访问时目录项 `hidden` 且不可聚焦。
- 不在走廊再增加三枚永久按钮，避免走廊现有入口继续膨胀。
- 痕迹页只增加一行：
  - `门前旁路：窥孔 / 失号龛 / 回返夹道（按已访问项显示）；你在门外改道 N 次。`
- 保持原有八张统计卡，不新增第九卡。

## 视觉与交互契约

- 三张正式素材均为 1536×1024 WebP，建议 Pillow quality 85，保留源 PNG。
- 新区域沿用 v29/v30 的 `.scene-branch`、标题、说明、位图、三动作布局与羽化语言；可以新增语义化类名，不复制一套完全不同的视觉系统。
- 桌面 1440×800：标题、说明、位图和三个动作全部落在首屏。
- 短桌面：首个动作无需滚动即可看见。
- 移动 390×844：标题、说明、位图和至少首个动作落在首屏；三动作单列，无横向溢出。
- 所有热点和动作均支持 Tab、Enter、Space，焦点清楚；图片 alt 要描述空间但不泄露选择答案。
- hover/focus 只做反馈，不自动换场。
- reduced-motion 缩短反馈，去掉位移/闪烁但不跳过状态文案。
- 静音状态、标题聚焦、滚动归顶、场景 veil 生命周期沿用现有契约。
